הגנה על דפים ב-Flask
Access Control

לומדים כיצד להגביל גישה לדפים — ולהבדיל בין מה שמשתמשים מחוברים רואים לבין אורחים

שלב 1 מתוך 8

🔒 מה זו הגנה על דפים?

לא כל דף באתר מיועד לכולם. בפרויקט שלנו, דף Restricted מיועד רק למשתמשים מחוברים. אורח שמנסה לגשת אליו — צריך לקבל דחייה.

זו אחת הדרישות הבסיסיות של כל אתר אמיתי: Authentication — זיהוי מי המשתמש, ו-Authorization — מה מותר לו לראות.

📋 מפת ההרשאות בפרויקט שלנו

דף אורח (Guest) משתמש מחובר
/ ✓ גישה ✓ גישה
/users ✓ גישה ✓ גישה
/login ✓ גישה ~ מיותר
/signup ✓ גישה ~ מיותר
/restricted ✗ חסום ✓ גישה
/logout ✗ לא רלוונטי ✓ גישה

💡 מה נלמד בלומדה זו?

  • כיצד Flask מזהה אם המשתמש מחובר (current_user)
  • הגנה על route בשרת — Python ב-main.py
  • הגנה על תפריט — Jinja2 ב-base.html
  • ההבדל בין הסתרה ויזואלית לחסימה אמיתית
  • כיצד להפנות משתמש חסום לדף אחר

👤 כיצד Flask יודע מי המשתמש?

בפרויקט שלנו משתמשים במשתנה גלובלי current_user שמאוחסן בשרת. הוא מתחיל כ-'Guest' ומשתנה לשם המשתמש אחרי login מוצלח.

  main.py — מחזור חיים של current_user
app.secret_key = "megido_secret_key"
valid_users = []

# ── מצב התחלתי: אורח ──
current_user = 'Guest'

# ── login מצליח: משתמש מחובר ──
if user in valid_users:
    current_user = user          # כעת: "alice" / "bob" / ...
    flash(" !נכנסת בהצלחה", "success")
    return redirect(url_for('home'))

# ── logout: חזרה לאורח ──
def logout():
    global current_user
    current_user = 'Guest'      # חזרה לברירת המחדל
    return redirect(url_for('home'))

🔍 הבדיקה הפשוטה

כדי לדעת אם מישהו מחובר, בודקים פשוט:

# מחובר?
if current_user != 'Guest':
    # כן — מותר לגשת

# אורח?
if current_user == 'Guest':
    # לא — לחסום
⚠️
הערה: בפרויקטים מקצועיים משתמשים ב-Session של Flask במקום משתנה גלובלי. אך הלוגיקה זהה — בדיקת מצב ההתחברות לפני הצגת התוכן.

🚧 הגנה על Route ב-main.py

זוהי ההגנה האמיתית — נמצאת בשרת. גם אם מישהו יקליד את הכתובת ישירות — ייחסם.

  main.py — הגנה על /restricted
@app.route('/restricted')
def restricted():
    if current_user != 'Guest':
        return render_template('restricted.html', user=current_user)
    else:
        return '<h1> access denied </h1>'

🔍 ניתוח הקוד

if current_user != 'Guest'
הבדיקה: האם יש משתמש מחובר? כל שם שאינו 'Guest' מחובר.
return render_template(...)
מחובר → מציגים את דף restricted.html עם שמו ("You now have access!")
return '<h1> access denied </h1>'
אורח → מחזירים HTML גולמי פשוט, ללא עיצוב ובלי base.html. הדף נראה "שבור" — זה מכוון!

🛠️ שיפור: redirect + Flash במקום HTML גולמי

הגרסה הנוכחית מחזירה HTML גולמי לאורחים. גרסה טובה יותר תפנה אותם לדף login עם הודעה:

❌ גרסה נוכחית
else:
    return '<h1>access denied</h1>'
    # HTML גולמי — לא נחמד!
✅ גרסה משופרת
else:
    flash("עליכם להתחבר תחילה", "error")
    return redirect(url_for('login'))
    # הפניה + הודעת Flash!
✨
הגרסה המשופרת מפנה לדף login ומציגה הודעת Flash — חוויה הרבה יותר טובה למשתמש.

🧭 הסתרת פריטי תפריט ב-base.html

הגנת השרת חוסמת גישה — אבל כדאי גם להסתיר מהתפריט קישורים שלא רלוונטיים. ב-base.html יש לוגיקת Jinja2 שמשנה את התפריט לפי מצב ההתחברות:

  templates/base.html — תפריט דינמי
<nav><ul>
    <!-- תמיד מופיע —>
    <li><a href="/"><i class="fas fa-home"></i> Home</a></li>
    <li><a href="/users"><i class="fas fa-users"></i> Users</a></li>

    <!-- הסתעפות לפי מצב התחברות —>
    {% if user == 'Guest' %}
        <!-- אורח רואה: Login + Signup -->
        <li><a href="/login"><i class="fas fa-sign-in-alt"></i> Login</a></li>
        <li><a href="/signup"><i class="fas fa-user-plus"></i> Signup</a></li>
    {% else %}
        <!-- מחובר רואה: Restricted + Logout -->
        <li><a href="/restricted"><i class="fas fa-user-secret"></i> Restricted</a></li>
        <li><a href="/logout"><i class="fas fa-sign-out-alt"></i> Logout</a></li>
    {% endif %}
</ul></nav>

🔍 ניתוח הלוגיקה

{% if user == 'Guest' %}
user הוא הפרמטר שנשלח מ-main.py בכל קריאה ל-render_template. אם הוא 'Guest' — אורח.
Login + Signup
אורח לא מחובר — מציגים את האפשרות להתחבר או להירשם.
Restricted + Logout
משתמש מחובר — מציגים גישה לדף המוגן ואפשרות להתנתק.
⚠️
חשוב להבין: הסתרה בתפריט היא לא הגנה! מישהו יכול להקליד ישירות /restricted בדפדפן. ההגנה האמיתית היא תמיד ב-main.py!

📚 סיכום — מה למדנו

  • current_user מתחיל כ-'Guest' ומשתנה לשם המשתמש אחרי login
  • הגנת שרת ב-main.py — if current_user != 'Guest' — חוסמת גישה אמיתית
  • הסתרת תפריט ב-base.html — {% if user == 'Guest' %} — משפרת UX בלבד
  • חסימה לבדה אינה מספיקה — תמיד שתי שכבות יחד
  • redirect + flash עדיפים על HTML גולמי כשחוסמים

🚀 משימת המשך

פותחים את הפרויקט ומנסים:


1. משנים את ה-else בפונקציית restricted() — במקום HTML גולמי, מפנים לדף login עם הודעת Flash

2. מוסיפים הגנה על /users — רק משתמשים מחוברים יוכלו לראות את רשימת המשתמשים

3. אתגר בונוס: מוסיפים דף חדש /admin שרק משתמש בשם "admin" יכול לגשת אליו — לא כל משתמש מחובר
