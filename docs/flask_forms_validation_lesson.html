📋 מה זה טופס ואיפה הוא ברויקט?

טופס (Form) הוא האמצעי שבו משתמשים שולחים מידע לאתר. בפרויקט שלנו יש שני טפסים מרכזיים:

🔑 login.html
<form method="post">
  <label for="username">Username:</label>
  <input type="text"
         id="username"
         name="username">
  <input type="submit"
         value="Login">
</form>

שדה אחד — שם משתמש. פשוט.

📝 signup.html
<form name="signupForm"
      onsubmit="return validateForm()"
      method="post">
  <!-- username, password, -->
  <!-- repeat-password,   -->
  <!-- email, phone        -->
</form>

5 שדות + אימות JavaScript.

🔑 שלוש תכונות חיוניות של <form>

method="post"
כיצד הנתונים נשלחים לשרת. POST = בגוף הבקשה, בלתי נראה. GET = ב-URL (פחות מאובטח לנתונים רגישים).
name="fieldName"
שם השדה שבו Python תשתמש כדי לשלוף את הערך: request.form['fieldName']
onsubmit="return validateForm()"
מריץ פונקציית JavaScript לפני שליחה. אם מחזירה false — השליחה נעצרת!
💡
מה נלמד: כיצד JavaScript מאמתת שדות לפני שליחה (צד לקוח), וכיצד Python מעבדת את הנתונים אחרי קבלתם (צד שרת).

🌐 GET לעומת POST — שתי דרכים לשלוח נתונים

כששולחים טופס, הנתונים נסעים מהדפדפן לשרת. יש שתי שיטות עיקריות:

GET — בקשת מידע
# URL נראה כך:
/login?username=alice

# ב-Python:
request.args['username']
⚠️
הנתונים גלויים ב-URL — לא מתאים לסיסמאות!
POST — שליחת נתונים
# URL נקי:
/login

# ב-Python:
request.form['username']
✅
הנתונים בגוף הבקשה — מתאים לטפסים עם נתונים אישיים.

🔀 כיצד Flask מטפל בשתי השיטות?

ה-route בפרויקט מציין את שתי השיטות המותרות:

  main.py
@app.route('/login', methods=['GET', 'POST'])
def login():
    global current_user
    if request.method == 'POST':
        # המשתמש שלח את הטופס
        user = request.form['username']
        ...
    return render_template('login.html', user=current_user, message='')
    # GET: פשוט מציג את הדף הריק
methods=['GET', 'POST']
מאפשרים לroute זה לקבל גם בקשות GET (פתיחת הדף) וגם POST (שליחת הטופס).
if request.method == 'POST'
בודקים את סוג הבקשה. כך אותה פונקציה מטפלת גם בהצגת הטופס וגם בעיבוד הנתונים.
return render_template(...)
אם GET (או אם POST נכשל) — מחזירים את הדף. שורה זו רצה רק כש-if לא מוציא return!

🐍 קריאת נתונים ב-Python עם request.form

אחרי שהמשתמש שולח טופס POST, כל הנתונים זמינים ב-request.form. הנה כיצד signup() שולפת את כל השדות:

  main.py — signup()
@app.route('/signup', methods=['GET', 'POST'])
def signup():
    if request.method == 'POST':
        username = request.form['username']   ← שם השדה ב-HTML
        password = request.form['password']
        email    = request.form['email']
        phone    = request.form['phone']

        # בדיקה: האם שם המשתמש כבר תפוס?
        if username in valid_users:
            flash(f"משתמש קיים {username}", "error")
            return render_template('signup.html', user=current_user,
                                    message='Username already taken.')

        # הכל תקין — שומרים ומפנים
        valid_users.append(username)
        flash(f"{username} נרשמת בהצלחה", "success")
        return redirect(url_for('login'))

    return render_template('signup.html', user=current_user, message='')

🧪 אימות צד לקוח — JavaScript

לפני שהנתונים בכלל מגיעים לשרת, JavaScript בודקת אותם בדפדפן. ב-signup.html:

  signup.html — הטריגר
<form name="signupForm"
      onsubmit="return validateForm()"
      method="post">

  <input type="text" id="username" name="username">
  <p id="username-error" class="error-message"></p>
  <!-- ↑ מיכל לשגיאה ליד כל שדה -->

  <input type="submit" value="Signup">
</form>

<!-- טעינת קובץ ה-JS -->
<script src="/static/script/signupvalidation.js"></script>

🔍 כיצד onsubmit עוצר את הטופס?

onsubmit="return validateForm()"
כשלוחצים Submit, הדפדפן מריץ validateForm(). אם היא מחזירה false — הטופס לא נשלח. אם true — נשלח.
id="username-error"
אלמנט ריק שממתין. JavaScript תכתוב לתוכו הודעת שגיאה אם הקלט לא תקין.
/static/script/signupvalidation.js
הקובץ נמצא בתיקיית static/script. זה הכתובת שFlask מגיש קבצים סטטיים ממנה.

📜 signupvalidation.js — הקוד המלא

קובץ זה מכיל את כל לוגיקת האימות בצד הלקוח. נפרק אותו לחלקים:

  static/script/signupvalidation.js — validateForm()
function validateForm() {
    var retValue = true;

    if (!validateUsername())   retValue = false;
    if (!validatePassword())   retValue = false;
    if (!validateRepeatPassword()) retValue = false;

    return retValue;  ← false = עצור שליחה!
}
  signupvalidation.js — validateUsername()
function validateUsername() {
    var username = document.getElementById("username").value;
    var usernameError = document.getElementById("username-error");
    var nameStartLetterRegex = /^[a-zA-Z]/;  ← Regex: מתחיל באות?

    if (username.length < 6) {
        usernameError.innerText = "name must be at least 6 characters long.";
        return false;
    } else if (!nameStartLetterRegex.test(username)) {
        usernameError.innerText = "name must start with a letter.";
        return false;
    } else {
        usernameError.innerText = "";  ← מנקה שגיאה ישנה
    }
    return true;
}
  signupvalidation.js — validatePassword()
function validatePassword() {
    var password = document.getElementById("password").value;
    var errorMessage = document.getElementById("password-error");

    var hasDigit   = /\d/;            ← הכיל ספרה?
    var hasSpecial = /[@$!%*?&]/;    ← הכיל תו מיוחד?

    if (!hasDigit.test(password)) {
        errorMessage.innerText = "Password must include at least one digit.";
        return false;
    } else if (!hasSpecial.test(password)) {
        errorMessage.innerText = "Password must include at least one special character.";
        return false;
    } else if (password.length < 7) {
        errorMessage.innerText = "Password must be at least 7 characters long.";
        return false;
    }
    errorMessage.innerText = "";
    return true;
}
  signupvalidation.js — validateRepeatPassword() + togglePassword()
function validateRepeatPassword() {
    var password       = document.getElementById("password").value;
    var repeatPassword = document.getElementById("repeat-password").value;
    var errorMessage   = document.getElementById("repeat-password-error");

    if (password != repeatPassword) {
        errorMessage.innerText = "Passwords do not match.";
        return false;
    }
    errorMessage.innerText = "";
    return true;
}

// כפתור "Show Password"
function togglePassword() {
    var input = document.getElementById("password");
    input.type = (input.type === "password") ? "text" : "password";
}

🆚 JS לעומת Python — למה שתיהן?

✅ JS — צד לקוח
  • מהיר — אין המתנה לשרת
  • מציג שגיאות ליד כל שדה
  • חוסך תעבורת רשת מיותרת

  • האחרונה לה סמכות
