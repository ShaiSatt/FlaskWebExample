💡 מה זה SQLite?

SQLite הוא מסד נתונים שמאוחסן בקובץ יחיד על הדיסק. הוא מובנה לתוך Python — לא צריך להתקין שום דבר נוסף.

1
קובץ אחד בלבד
כל מסד הנתונים יושב בקובץ users.db בתיקיית הפרויקט. אפשר לשלוח, לגבות, למחוק — כמו קובץ רגיל.
2
מובנה ב-Python
ספריית sqlite3 מגיעה עם כל התקנת Python. אין pip install. פשוט import sqlite3 ומתחילים.
3
שפת SQL
מתקשרים עם ה-DB בעזרת שפת SQL — שפה סטנדרטית לניהול נתונים. נלמד אותה צעד אחר צעד.

💡 מה נלמד בלומדה זו?

  • מה זה SQLite ולמה הוא מתאים לפרויקטים קטנים
  • הקוד המלא של create_table() — פירוק שלב אחר שלב
  • פתיחת חיבור ל-DB: connect() ו-cursor()
  • פקודת CREATE TABLE IF NOT EXISTS — ומה כל מילה עושה
  • הגדרת 4 שדות: שם, סוג נתון ומגבלות
  • שמירה וסגירה: commit() ו-close()

📋 הקוד המלא — create_table()

הנה הפונקציה המלאה מ-main.py שאחראית על יצירת הטבלה. בשקפים הבאים נפרק כל שורה:

  main.py
import sqlite3                          ← חלק א׳: ייבוא

def create_table():                       ← חלק ב׳: הפונקציה
    conn = sqlite3.connect('users.db')  ← פתיחת חיבור
    c = conn.cursor()                    ← יצירת cursor
    c.execute('''
        CREATE TABLE IF NOT EXISTS users
        (
            username TEXT    PRIMARY KEY  NOT NULL,
            password TEXT                  NOT NULL,
            email    TEXT                  NOT NULL,
            phone    TEXT
        )''')
    conn.commit()                         ← שמירה לדיסק
    conn.close()                          ← סגירת חיבור

create_table()                           ← חלק ג׳: קריאה מיידית
💡
בשקפים הבאים נפרק כל צבע בנפרד: סגול = חיבור ל-DB, כחול = הגדרת הטבלה, ירוק = שמירה וסגירה, כתום = קריאה לפונקציה.

לפני שאפשר לעשות כל פעולה על ה-DB, חייבים לפתוח חיבור אליו:

  main.py
def create_table():
    conn = sqlite3.connect('users.db')
    c = conn.cursor()
    ...
sqlite3.connect('users.db')
פותח חיבור לקובץ. אם הקובץ users.db כבר קיים — פותח אותו. אם לא קיים — יוצר אותו אוטומטית. מחזיר אובייקט חיבור שאנחנו שומרים ב-conn.
conn.cursor()
יוצר cursor — כמו "עט" שכותב ומוחק בתוך ה-DB. דרכו שולחים את כל פקודות ה-SQL. המוסכמה בפרויקט: לקרוא לו c.

📂 היכן נוצר users.db?

SQLite יוצר את הקובץ באותה תיקייה שממנה מריצים את main.py:

FlaskWebExample/
├── main.py
├── users.db        ← נוצר כאן בהרצה ראשונה
├── templates/
└── static/
💡
הרצה ראשונה: users.db לא קיים ← connect() יוצר קובץ ריק ← create_table() מוסיפה את הטבלה. הרצה שנייה: users.db קיים ← connect() פותח אותו ← IF NOT EXISTS דואג שהטבלה לא תיכתב שוב.

🏗️ שלב 2 — פקודת CREATE TABLE

לאחר פתיחת החיבור, שולחים פקודת SQL ליצירת הטבלה:

  main.py — שאילתת ה-SQL
    c.execute('''
        CREATE TABLE IF NOT EXISTS users
        (
            ...
        )
    ''')
c.execute( ... )
שולח שאילתת SQL ל-DB דרך ה-cursor. מקבל מחרוזת עם פקודת SQL. ה-''' ''' מאפשר כתיבה בכמה שורות.
CREATE TABLE
פקודת SQL שיוצרת טבלה חדשה. SQL כתוב באותיות גדולות במוסכמה — Python לא מחייב זאת, אבל זה קריא יותר.
IF NOT EXISTS
קריטי! בלעדיו: בכל הרצת שרת תנסה ליצור טבלה שכבר קיימת — שגיאה. עם IF NOT EXISTS: "אם כבר קיים — תדלג".
users
שם הטבלה. כמו שם לרשימה. יכול להיות כל שם — בפרויקט בחרנו users כי זה הגיוני.

📊 מה ההבדל בין טבלה לגיליון אקסל?

טבלת SQL דומה לגיליון — שורות ועמודות. ההבדל: כל עמודה מוגדרת מראש עם שם, סוג נתון ומגבלות.

🗃️ users  — טבלה ריקה לאחר CREATE TABLE
usernamepasswordemailphone
אין שורות עדיין — הטבלה ריקה
💡
CREATE TABLE מגדיר רק את המבנה — העמודות. השורות (הנתונים עצמם) יתווספו מאוחר יותר עם INSERT, שנלמד בלומדה הבאה.

📐 שלב 3 — הגדרת השדות (עמודות)

בתוך הסוגריים של CREATE TABLE מגדירים כל עמודה — שם, סוג נתון ומגבלות:

  SQL — הגדרת עמודות
        (
            username TEXT    PRIMARY KEY  NOT NULL,
            password TEXT                  NOT NULL,
            email    TEXT                  NOT NULL,
            phone    TEXT
        )

🔧 מבנה הגדרת עמודה

   שם_עמודה    סוג_נתון   מגבלות
   ─────────    ─────────   ────────────────────────
   username     TEXT        PRIMARY KEY  NOT NULL
   password     TEXT        NOT NULL
   email        TEXT        NOT NULL
   phone        TEXT        (ללא מגבלות — אופציונלי)

🔢 סוגי נתונים ב-SQLite

TEXT
טקסט — מחרוזת באורך חופשי. השדה הנפוץ ביותר. מתאים לשמות, סיסמאות, אימייל, מספרי טלפון.
INTEGER
מספר שלם — לא בשימוש בפרויקט שלנו, אבל נפוץ למזהים (ID) ולספירות.
REAL
מספר עשרוני — לא בשימוש כאן. מתאים למחירים, ציונים.
💡
מספר טלפון כ-TEXT ולא כ-INTEGER? מכיוון שמספרי טלפון מתחילים ב-0 (כמו 050-...) ועשויים להכיל מקפים — TEXT שומר אותם בדיוק כפי שהוזנו.

🔒 מגבלות שדות — Constraints

מגבלות קובעות אילו ערכים מותרים בכל עמודה. בפרויקט שלנו יש שלוש מגבלות:

NOT NULL
שדה חובה — SQLite לא יאפשר להכניס שורה בלי ערך בשדה הזה. username, password ו-email הם NOT NULL — חייבים להיות ממולאים.
PRIMARY KEY
מזהה ייחודי — ערך זה חייב להיות ייחודי בכל הטבלה. אי אפשר שיהיו שני משתמשים עם אותו username. גם כולל NOT NULL אוטומטית.
(ללא מגבלה)
שדה אופציונלי — phone ללא מגבלות. ניתן להכניס שורה בלי טלפון. הערך יהיה NULL (=ריק) ב-DB.

📊 ויזואליזציה — ניסיון כניסת שורה כפולה

🗃️ users
username PK password NN email NN phone
alice pass123! alice@school.ac.il 050-1234567
bob42 b0b$ecure bob@email.com NULL
alice newpass alice2@x.com —

↑ שורה שלישית נדחית — alice כבר קיים כ-PRIMARY KEY

💾 commit() ו-close() — סיום נכון

    conn.commit()   ← שומר שינויים מהזיכרון לקובץ הדיסק
    conn.close()    ← שחרור החיבור — חובה לאחר הסיום
conn.commit()
שומר לדיסק. בלי commit, ה-CREATE TABLE קיים רק בזיכרון ויאבד. אחרי commit — הטבלה כתובה בקובץ users.db.
conn.close()
סוגר חיבור. משחרר את הקובץ. תמיד לסגור בסוף — לא לפני commit!

כי IF NOT EXISTS מהיר יותר

🏁 סיכום הלומדה

  • SQLite — מסד נתונים בקובץ יחיד, מובנה ב-Python, ללא התקנה
  • sqlite3.connect('users.db') — פותח או יוצר את הקובץ
  • conn.cursor() — יוצר "עט" לשליחת שאילתות SQL
  • CREATE TABLE IF NOT EXISTS users (...) — יוצר טבלה רק אם לא קיימת
  • 4 עמודות: username (PK+NN), password (NN), email (NN), phone (אופציונלי)
  • TEXT — סוג הנתון לכל השדות בפרויקט
  • PRIMARY KEY — ייחודיות. NOT NULL — חובה למלא
  • conn.commit() ← שמירה. conn.close() ← סגירה
  • create_table() נקראת מיד בטעינת main.py — לפני כל route

🚀 משימת המשך
