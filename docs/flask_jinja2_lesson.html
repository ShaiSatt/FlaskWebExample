✨ מה זו Jinja2?

HTML הוא שפה סטטית — הוא לא יודע "לחשוב". לא ניתן לכתוב בו "אם המשתמש מחובר, הצג כפתור logout". לשם כך Flask משתמש ב-Jinja2 — שפת תבניות שמוסיפה לוגיקה לתוך ה-HTML.

Jinja2 רצה בשרת, לפני שה-HTML נשלח לדפדפן. הדפדפן רואה HTML רגיל — הוא אינו מודע ל-Jinja2.

Jinja2 — בשרת
<h4>Welcome {{ user }}!</h4>
{% if user == 'Guest' %}
  <a href="/login">Login</a>
{% else %}
  <a href="/logout">Logout</a>
{% endif %}
HTML שמגיע לדפדפן
<!-- אם user = 'alice' -->
<h4>Welcome alice!</h4>
  <a href="/logout">Logout</a>


<!-- כל הjinja2 נעלם -->

🔑 שלוש סוגי תגיות Jinja2

{{ ... }}
הצגת ערך — מחליף ביטוי בתוצאה שלו. למשל {{ user }} הופך ל-"alice".
{% ... %}
פקודת לוגיקה — if, for, block, extends, with, וכו'. לא מוציאה טקסט ישיר.
{# ... #}
הערה — לא מוצגת ב-HTML הסופי. שימושית לתיעוד בקוד.
💡
נלמד בלומדה זו: כל סוגי התגיות — עם דוגמאות מהקוד האמיתי מהפרויקט שלנו.

🟠 {{ }} — הצגת משתנים

שתי סוגריים מסולסלות מציגות את הערך של משתנה שנשלח מ-Python. זה הפשוט ביותר בJinja2.

  שימושים ב-{{ }} מהפרויקט
── base.html ──
<h4> Welcome {{ user }} !</h4>
   ↑ Python שולחת: render_template('...', user=current_user)

── base.html — Flash message ──
<div class="flash-box {{ category }}">
    {{ message }}
</div>
   ↑ category = "success" / "error"
     message  = "!נכנסת בהצלחה"

── restricted.html ──
<p>{{ user }} You now have access!</p>

── users.html — בתוך לולאה ──
{% for user in users %}
    <td>{{ user }}</td>
{% endfor %}

🔗 כיצד המשתנה מגיע מ-Python?

render_template('home.html', user=current_user)
Python מעבירה את current_user תחת השם user. בתבנית — {{ user }} = הערך.
render_template("users.html", user=current_user, users=valid_users)
ניתן להעביר כמה משתנים — כל אחד נגיש בשמו ב-{{ }}.
render_template('login.html', user=current_user, message='')
גם מחרוזות ריקות מועברות. login.html מציגה שגיאה רק אם message לא ריק.
⚠️
שים לב: {{ user }} בתבנית מתייחס לפרמטר ששולח render_template, לא למשתנה current_user בPython. השם חייב להתאים!

🟣 {% if %} — תנאים

תגית הלוגיקה החזקה ביותר — מציגה חלקים שונים של HTML בהתאם לתנאי. בפרויקט יש שימוש כפול:

  base.html — תפריט דינמי
{% if user == 'Guest' %}
    <li><a href="/login">Login</a></li>
    <li><a href="/signup">Signup</a></li>
{% else %}
    <li><a href="/restricted">Restricted</a></li>
    <li><a href="/logout">Logout</a></li>
{% endif %}
  base.html — Flash messages
{% with messages = get_flashed_messages(with_categories=true) %}
    {% if messages %}   ← בודק: יש הודעות בכלל?
        {% for category, message in messages %}
            <div class="flash-box {{ category }}">
                {{ message }}
            </div>
        {% endfor %}
    {% endif %}
{% endwith %}

📐 מבנה מלא עם elif

Jinja2 תומכת גם ב-{% elif %} לתנאים מרובים:

{% if user == 'Guest' %}
    <p>אינכם מחוברים</p>
{% elif user == 'admin' %}
    <p>שלום מנהל!</p>
{% else %}
    <p>שלום {{ user }}!</p>
{% endif %}
{% if messages %}
אם messages היא רשימה ריקה — Python מחשיב אותה כ-False. כך אנחנו בודקים "יש בכלל הודעות?" בלי לבדוק אורך ידנית.

🔁 {% for %} — לולאות

לולאת for מאפשרת ליצור HTML חוזר עבור כל פריט ברשימה. users.html משתמשת בה כדי לבנות שורת טבלה לכל משתמש:

  templates/users.html
<table class="userstable">
    <thead><tr><th>Username</th></tr></thead>
    <tbody>
        {% for user in users %}
        <tr>
            <td>{{ user }}</td>
        </tr>
        {% endfor %}
    </tbody>
</table>

📊 מה נוצר בפועל?

אם valid_users = ['alice', 'bob', 'charlie'], הלולאה מייצרת:

קוד Jinja2
{% for user in users %}
<tr><td>{{ user }}</td></tr>
{% endfor %}
HTML שנוצר
<tr><td>alice</td></tr>
<tr><td>bob</td></tr>
<tr><td>charlie</td></tr>
תצוגה בדפדפן:
Username
alice
bob
charlie
{% for category, message in messages %}
לולאה על רשימת צמדים — כל איטרציה מפרקת אוטומטית ל-category ו-message. כך base.html מציגה הודעות Flash.

📦 {% with %} ו-{% block %}

שתי פקודות מיוחדות שמשמשות בפרויקט שלנו לצרכים שונים:

  base.html — שימוש ב-{% with %}
{% with messages = get_flashed_messages(with_categories=true) %}
    {% if messages %}
        {% for category, message in messages %}
            <div class="flash-box {{ category }}">{{ message }}</div>
        {% endfor %}
    {% endif %}
{% endwith %}
{% with messages = ... %}
יוצר משתנה זמני messages שקיים רק בתוך הבלוק. מחוץ ל-endwith — המשתנה לא קיים. שימושי לקריאת קוד.
  base.html + home.html — {% block %}
── base.html: מגדיר "חלון" ──
<title>{% block title %}Flask App{% endblock %}</title>

<main class="container">
    {% block content %}
    {% endblock %}
</main>

── home.html: ממלא את "החלון" ──
{% extends "base.html" %}

{% block title %}Home{% endblock %}

{% block content %}
    <h2>Home Page</h2>
{% endblock %}
{% block name %} ברירת מחדל {% endblock %}
ב-base.html: מגדיר אזור עם תוכן ברירת מחדל ("Flask App" בכותרת). אם דף-בן לא מגדיר בלוק זה — ברירת המחדל מוצגת.

🔗 {{ url_for() }} — יצירת קישורים דינמיים

במקום לכתוב כתובות URL ישירות, Jinja2 מאפשרת ליצור אותן לפי שם הפונקציה ב-main.py. זה מונע שגיאות אם כתובת משתנה:

  שימוש ב-url_for בפרויקט
── base.html: טעינת CSS ──
<link rel="stylesheet"
      href="{{ url_for('static', filename='css/style.css') }}">
  ↑ Flask מחזיר: /static/css/style.css

── main.py: redirect ──
return redirect(url_for('home'))
  ↑ 'home' = שם הפונקציה, לא ה-URL

return redirect(url_for('login'))
  ↑ Flask מחפש את הפונקציה login() ומחזיר /login

🔍 למה url_for ולא /static/css/style.css ישירות?

url_for('static', filename='css/style.css')
Flask יוצר את הנתיב הנכון תמיד — גם אם האפליקציה רצה בנתיב שאינו root, או שמבנה התיקיות משתנה.
url_for('home')
Flask מחפש את ה-route עם @app.route ששייך לפונקציה home(). אם ה-URL ישתנה מ-'/' למשהו אחר — הקוד עדיין יעבוד!
✅
כלל טוב: בתוך תבניות — תמיד {{ url_for('static', ...) }} לקבצים סטטיים. ב-Python — תמיד url_for('function_name') ב-redirect.

📚 סיכום — מה למדנו

  • {{ }} — מציגה ערך משתנה שנשלח מ-render_template
  • {% if %} / {% elif %} / {% else %} / {% endif %} — הסתעפות לוגית
  • {% for x in list %} / {% endfor %} — לולאה ליצירת HTML חוזר
  • {% with var = expr %} / {% endwith %} — משתנה זמני מקומי
  • {% block %} / {% endblock %} — חלון שדפי בן ממלאים
  • {% extends %} — ירושת תבנית אם
  • {{ url_for() }} — יצירת URL מבוסס שם פונקציה

🚀 משימת המשך

פותחים את הפרויקט ומנסים:


1. מוסיפים ב-users.html לולאה שמציגה גם מספר סידורי לצד כל שם — {% for i, user in enumerate(users) %} בJinja2 כתוב: {{ loop.index }}

2. מוסיפים ב-base.html הודעה ייחודית כשהרשימה ריקה: {% if users %} ... {% else %}<p>אין משתמשים{% endif %}

3. אתגר בונוס: מוסיפים ב-base.html תגית הערה Jinja2 {# #} שמסבירה את הלוגיקה של ה-if — ומוודאים שהיא לא מופיעה ב-DevTools
