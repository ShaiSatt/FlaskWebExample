🗺️ ניתוב — Routes

Route הוא הקשר בין כתובת URL לפונקציה Python. ה-decorator @app.route הוא מה שיוצר את הקשר הזה.

URLMethodsפונקציהתיאור / GET home() דף הבית — מציג home.html /login GETPOST login() GET: מציג טופס. POST: מעבד התחברות /logout GET logout() מאפס current_user ל-Guest ומפנה לבית /restricted GET restricted() מוגן — רק למשתמשים מחוברים /signup GETPOST signup() GET: מציג טופס. POST: רושם משתמש חדש /users GET show_users() מציג רשימת כל המשתמשים הרשומים

// כיצד decorator עובד?

# ← decorator: רץ לפני הגדרת הפונקציה
@app.route('/')
def home():
    return render_template('home.html', user=current_user)

# ← route עם שיטות מרובות
@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        ...   ← עיבוד טופס
    return render_template('login.html', user=current_user, message='')
@app.route('/')
decorator — "עוטף" את הפונקציה ורושם אותה אצל Flask. כשמגיעה בקשה ל-'/', Flask ידע לקרוא ל-home().
methods=['GET', 'POST']
ברירת מחדל: רק GET. עבור routes שמקבלים טפסים — חייבים להוסיף POST. בלעדיו Flask יחזיר שגיאה 405.

redirect אחרי POST
Post-Redirect-Get (PRG) — אחרי עיבוד טופס POST, תמיד מפנים עם redirect. כך רענון הדף לא ישלח את הטופס שוב!

// כיצד url_for() יוצר URLs?

url_for('home')      → '/'
url_for('login')     → '/login'
url_for('show_users') → '/users'
# ↑ שם הפונקציה, לא ה-URL!
# שימושי כי: אם נשנה את ה-URL ב-route — url_for יעדכן אוטומטית

url_for('static', filename='css/style.css') → '/static/css/style.css'

🧩 שלושה דפוסים חוזרים ב-main.py

כמעט כל route בפרויקט עונה על אחד משלושה דפוסים. זיהויים יעזור לכם לכתוב routes חדשים:

דפוס 1 — GET בלבד: מציג מידע
@app.route('/')
def home():
    return render_template('home.html', user=current_user)

# גם: /users, /restricted (עם בדיקת הרשאה)

פשוט — מקבל בקשה, מחזיר HTML.

דפוס 2 — GET+POST: טופס
@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        # ← עיבוד נתוני הטופס
        user = request.form['username']
        if user in valid_users:
            flash("נכנסת בהצלחה", "success")
            return redirect(url_for('home'))
        flash(f"משתמש לא קיים {user}", "error")
    # ← GET: מציג את הטופס הריק (גם אם POST נכשל)
    return render_template('login.html', user=current_user, message='')

GET → מציג. POST → מעבד → redirect אם הצלחה, render אם כישלון.

דפוס 3 — פעולה + redirect
@app.route('/logout')
def logout():
    global current_user
    flash(f"התנתקת בהצלחה! {current_user}", "success")
    current_user = 'Guest'
    return redirect(url_for('home'))

📋 סיכום — מה למדנו

  • מבנה הפרויקט: main.py + templates/ + static/ — כל אחד ותפקידו
  • main.py בנוי מ-4 חלקים: ייבואים, אתחול, routes, הפעלה
  • decorator @app.route קושר URL לפונקציה Python
  • methods=['GET','POST'] — GET מציג, POST מעבד נתונים
  • render_template מחזיר HTML, redirect שולח לURL אחר
  • url_for מייצר URL לפי שם פונקציה — עדיף על URL ידני
  • global current_user — נדרש בפונקציות שמשנות משתנה גלובלי
  • valid_users בזיכרון בלבד — בפרויקט אמיתי: Database
